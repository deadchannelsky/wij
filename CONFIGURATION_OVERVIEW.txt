================================================================================
QUICK EDITING GUIDE
================================================================================

TO EDIT THE ASSISTANT PROMPT:
  1. Open server/config/assistant.config.json
  2. Find: "assistant": { "systemPrompt": "..."
  3. Edit the systemPrompt string
  4. Save and commit
  5. Restart: sudo systemctl restart whereisjason

TO EDIT JASON'S INFORMATION:
  1. Open server/config/assistant.config.json
  2. Find: "context": { ... }
  3. Edit the fields (name, title, company, etc.)
  4. Save and commit
  5. Restart: sudo systemctl restart whereisjason

TO EDIT LLM ENDPOINT & AUTH:
  1. Open server/config/assistant.config.json
  2. Find: "llm": { ... }
  3. Edit endpoint, model, apiKey, useApiKey as needed
  4. Save and commit (or use environment variables for secrets)
  5. Restart: sudo systemctl restart whereisjason

================================================================================
CONFIGURATION PRIORITY
================================================================================

When both config file and environment variables exist:

  Environment Variables (.env)
        ↓ (override)
  server/config/assistant.config.json
        ↓ (fallback)
  Code defaults

This means:
• Use config file for normal settings
• Use environment variables (.env) for secrets (API keys)

================================================================================
ENVIRONMENT VARIABLE OVERRIDES
================================================================================

For production, keep secrets in environment variables:

  LLM_ENDPOINT=http://192.168.1.50:11434
  LLM_MODEL=neural-chat
  LLM_API_KEY=sk-xxxxxxxxxxxxx

Set these in .env.production and they will override config.json values.
